# 企业背调与风险预警

> 本 Skill 为「职途星」专家的核心能力模块，用于对目标公司进行全面背调。覆盖法人/高管征信、劳动仲裁与欠薪违法、薪资发放风险、股权异常、行业竞争力等维度，帮助求职者入职前避坑。

## 触发时机（两种场景）

### 场景一：用户有明确意向公司（原有逻辑）

💡用户提供目标公司名称 **且** 通过岗位硬性条件适配判断后执行。

#### 前置条件（必须同时满足）

1. **用户已提供目标公司名称**：无公司名则无法背调
2. **已完成JD解析与匹配度分析**：必须先知道岗位要求是什么
3. **已通过岗位硬性条件适配判断**：用户满足JD的硬性条件（学历/年限/证书/必须技能等）

### 场景二：智能推荐匹配公司（🆕 新增）

当用户未提供目标公司，但简历信息充分（已通过简历诊断提取完整职业画像）时：

1. **提取用户职业画像**：从简历中提取行业、职能、职级、核心技能、薪资范围
2. **筛选匹配公司**：结合行业知识和用户画像，推荐3-5家匹配的公司
3. **逐家背调**：对推荐的公司逐一执行完整背调流程
4. **生成投递价值排序**：基于背调结果+匹配度，输出投递优先级建议

> ⚠️ **注意**：场景二的背调结果需标注「基于公开信息推断，建议用户自行验证」的免责说明。

#### 什么情况下跳过背调

以下情况**不执行背调**，直接跳过：

| 情况 | 原因 | 替代动作 |
|------|------|---------|
| 用户未提供公司名称且简历信息不足以推断匹配公司 | 无法确定背调对象 | 交付时温馨提示"提供公司名可做背调" |
| 用户未提供JD（场景一） | 无法判断是否值得投递 | 交付时温馨提示"提供JD可做精准匹配+背调" |
| **不满足JD硬性条件**（场景一） | 投递都不会过初筛，背调无意义 | 明确告知用户差距，建议调整投递方向 |

### 硬性条件适配判断标准

| 判断维度 | 不满足的表现 | 判断方式 |
|---------|------------|---------|
| 学历要求 | JD要求"硕士及以上"，用户为本科 | 直接对比 |
| 工作年限 | JD要求"5年以上"，用户只有2年 | 直接对比 |
| 必须证书 | JD要求"持有CPA"，用户无此证书 | 直接对比 |
| 必须技能 | JD要求"熟练使用Python"，用户简历无此技能 | 直接对比 |
| 行业经验 | JD要求"有金融行业经验"，用户无此背景 | 直接对比 |

> **注意**：只有"硬性条件"不满足才跳过背调。"加分项"不满足不影响——加分项本来就是锦上添花。

### 特殊场景：路径C（面试阶段）

用户已进入面试环节时，说明已通过公司初筛，**背调无条件执行**，帮助用户判断公司是否值得入职。

## 为什么要做企业背调

求职是双向选择。企业背调的核心目的是帮求职者在入职前掌握关键信息，规避以下**求职者最关心的风险**：

1. **💰 薪资发放风险**（🆕 最高优先级）：公司是否有拖欠薪资/降薪/停发社保的劣迹？资金链是否断裂？发薪是否稳定？
2. **⚖️ 劳动仲裁与欠薪违法**（🆕 最高优先级）：公司是否有大量劳动仲裁？是否被员工起诉欠薪？是否被列入欠薪"黑名单"？
3. **🔍 法人/高管征信**（🆕 最高优先级）：法定代表人和核心高管是否有失信记录？征信是否异常？是否被限制高消费？
4. **📉 法律风险**：公司是否有经营异常、行政处罚、偷税漏税等违法行为？
5. **🏢 经营风险**：公司业务是否在走下坡路？是否有大规模裁员迹象？营收是否持续下滑？
6. **🔗 股权风险**：股权架构是否异常（如频繁变更法人、大量股权出质/冻结）？

## 背调执行流程（🆕 升级为8步）

### Step 1 — 基础工商信息核实（必做）

- 调用 `get_company_by_query` → 锁定目标公司（精确匹配公司全称）
- 调用 `get_company_registration_info` → 获取：注册资本（重点关注实缴非认缴）、成立日期、经营状态（存续/注销/吊销）、法定代表人
- 调用 `get_company_profile` → 获取：公司简介、业务模式、行业分类、人员规模、参保人数

> 🔍 **求职者视角**：注册资本是"认缴"还是"实缴"？实缴少意味着抗风险能力弱；参保人数远小于宣称员工数，说明社保缴纳不规范。

### Step 2 — 法人/高管诚信与征信深度审查（必做）🆕 重点增强

- 调用 `get_key_personnel` → 获取核心高管名单（法定代表人、董事长、总经理、监事）
- 调用 `get_contact_info` → 获取公司公开联系方式（空号/停机为异常信号）

**对法定代表人，逐项审查以下征信维度**：

| 审查维度 | 查询方式 | 风险信号 |
|---------|---------|---------|
| 🚫 失信被执行人（老赖） | `verify_company_accuracy` 交叉验证 | 法定代表人被列为失信被执行人 |
| 🚫 限制高消费 | `verify_company_accuracy` + 司法风险分析 | 有"限消令"记录 |
| 🚫 被执行人记录 | 分析司法风险数据 | 近3年有被执行记录 |
| ⚠️ 征信异常 | 交叉比对工商与司法数据 | 多家公司法人异常、频繁变更 |
| ⚠️ 违法犯罪记录 | 综合司法数据分析 | 有刑事/行政处罚记录 |
| ⚠️ 其他任职风险 | 查看法人名下其他公司 | 名下多家公司经营异常/注销 |

**对核心高管（董事长、总经理、监事），逐项审查**：

| 审查维度 | 查询方式 | 风险信号 |
|---------|---------|---------|
| 个人失信 | 交叉验证 | 高管被列为失信被执行人 |
| 历史关联风险 | 查看高管曾任职企业 | 曾任职企业大量注销/被处罚 |
| 行业口碑 | 结合公开信息评估 | 频繁跳槽、行业负面评价 |

- ⚠️ **如果法定代表人被列为失信被执行人/限制高消费，必须立即向用户发出 🔴 高风险警告并中断流程**
- ⚠️ **如果核心高管中有 ≥2 人存在失信/被执行记录，同样触发高风险警告**

### Step 3 — 劳动仲裁与欠薪违法专项审查（必做）🆕 重点增强

#### 3.1 劳动仲裁审查

- 调用 `get_company_registration_info` 查看所有司法案件
- 逐条分析案件类型，筛选出**劳动仲裁案件**
- 重点关注以下指标：

| 指标 | 风险等级 | 含义 |
|------|---------|------|
| 近1年劳动仲裁 ≥5起 | 🔴 极高风险 | 公司存在系统性问题，员工关系恶劣 |
| 近1年劳动仲裁 3-4起 | 🔴 高风险 | 用工纠纷频繁，需高度警惕 |
| 近1年劳动仲裁 1-2起 | 🟡 需关注 | 个例或趋势开始 |
| 近1年劳动仲裁 0起 | 🟢 正常 | 暂无用工纠纷公开记录 |
| 累计劳动仲裁 ≥10起 | 🔴 极高风险 | 长期存在用工问题 |

- 进一步分析每起劳动仲裁的**原告方身份**：是员工起诉公司（公司败诉概率大）还是公司起诉员工？
- 统计仲裁**案由类型**：欠薪、违法解除合同、工伤赔偿、社保纠纷各占比多少？

#### 3.2 欠薪违法专项审查

- 从司法数据中筛选**与薪资拖欠直接相关的案件**：
  - 劳动合同纠纷（重点关注：追索劳动报酬）
  - 劳务派遣合同纠纷（重点关注：拖欠工资）
  - 社会保险纠纷（重点关注：欠缴社保/公积金）
- 调用 `get_company_registration_info` 查看是否有**行政处罚**涉及欠薪
- 查看是否有被**人社部门列入拖欠工资"黑名单"**的记录
- 查看 **裁判文书**中是否存在"拖欠工资""未足额支付""欠薪"等关键词

#### 3.3 经济纠纷审查

- 筛选非劳动类经济案件：合同纠纷、债务纠纷、股权纠纷
- 关注**公司作为被告**的案件数量（被起诉多意味着信任度低）
- 关注**涉案金额**（大额纠纷可能威胁公司现金流）

### Step 4 — 薪资发放风险深度评估（必做）🆕 全新步骤

这是求职者最关心的维度，必须从多个角度交叉验证：

#### 4.1 资金链健康度评估

| 评估维度 | 查询方式 | 风险信号 |
|---------|---------|---------|
| 注册资本实缴比例 | `get_company_registration_info` | 实缴为0或远低于认缴 |
| 财务数据 | `get_financial_data` | 营收下滑、净利润亏损、纳税大幅减少 |
| 社保参保人数 | `get_company_profile` | 参保人数骤降（可能裁员或断缴社保） |
| 对外投资 | `get_external_investments` | 大规模减持/退出对外投资（套现自救） |

#### 4.2 发薪稳定性交叉验证

- 综合劳动仲裁案由，统计"追索劳动报酬"类案件占比
- 查看裁判文书中是否存在"拖欠X个月工资""未按时足额支付"等描述
- 查看是否存在**集体性欠薪**（多名员工同时起诉）
- 查看行业动态：同行业公司近期是否普遍降薪/裁员

#### 4.3 社保公积金缴纳合规性

- 查询公司参保人数 vs 宣称员工人数：
  - 参保人数 < 宣称人数的70% → 🟡 需关注
  - 参保人数 < 宣称人数的50% → 🔴 高风险（大量员工未缴社保）
- 查看是否有社保纠纷相关的劳动仲裁

#### 4.4 裁员/业务收缩信号

| 信号 | 查询方式 | 含义 |
|------|---------|------|
| 分支机构大量注销 | `get_branches` | 业务收缩，可能伴随裁员 |
| 参保人数骤降 | `get_company_profile` | 大量员工离职或裁员 |
| 工商变更：经营场所缩小 | `get_change_records` | 成本压缩信号 |
| 招聘冻结信号 | 行业公开信息 | 官网/招聘平台停止更新 |

### Step 5 — 股权架构审查（必做）

- 调用 `get_shareholder_info` → 获取股东结构、持股比例
- 调用 `get_actual_controller` → 穿透股权，识别实际控制人
- 调用 `get_beneficial_owners` → 识别受益所有人（UBO）
- 重点关注：
  - 股权是否频繁变更（3年内法人变更≥2次为异常信号）
  - 是否有大量股权出质/冻结
  - 实际控制人是否隐匿（无法穿透识别）
  - 是否存在"代持"嫌疑（法人非实际控制人）

### Step 6 — 经营与行业竞争力评估（必做）

- 调用 `get_company_profile` 了解公司业务和产品定位
- 调用 `get_financial_data` 获取财务数据（营收、利润、纳税情况）
- 调用 `get_external_investments` 了解公司对外投资布局
- 调用 `get_listing_info` 查看是否上市/拟上市
- 结合行业知识评估：公司在同行中处于什么位置？是头部/腰部/尾部？
- 查看人员规模变化趋势（增长？稳定？收缩？）

### Step 7 — 其他异常信号扫描（必做）

- 调用 `get_change_records` 查看工商变更记录——频繁变更为异常信号
- 调用 `get_branches` 查看分支机构——大规模注销分支机构=业务收缩
- 调用 `get_contact_info` 查看联系方式——空号/停机/地址异常为风险信号
- 调用 `get_annual_reports` 查看年报——未按时公示年报=管理不规范
- 调用 `get_tax_invoice_info` 查看税务状态——非正常户/欠税为严重风险

### Step 8 — 综合风险评级与投递建议（必做）

基于前7步的所有数据，综合判定风险等级并给出投递建议。

## 风险等级判定标准（🆕 扩增版）

### 🔴 高风险（任一触发即判定）

| 触发条件 | 所属维度 |
|---------|---------|
| 法定代表人被列为失信被执行人 | Step 2 法人征信 |
| 法定代表人有"限制高消费"记录 | Step 2 法人征信 |
| 核心高管中 ≥2 人有失信/被执行记录 | Step 2 高管征信 |
| 近1年劳动仲裁 ≥3起，且其中 ≥50% 为"追索劳动报酬"类 | Step 3 劳动仲裁 |
| 存在被列入"拖欠工资黑名单"记录 | Step 3 欠薪违法 |
| 有集体性欠薪诉讼（≥3名员工同时起诉） | Step 3 欠薪违法 |
| 参保人数 < 宣称员工数的50% | Step 4 薪资风险 |
| 财务数据显示营收连续下滑+净利润持续亏损 | Step 4 薪资风险 |
| 股权大面积冻结/出质（≥30%股权受限） | Step 5 股权 |
| 公司经营状态为"吊销"或"注销" | Step 1 工商 |

> **🔴 高风险动作规则**：发现任一高风险条件时，**必须立即中断当前流程**，向用户展示风险详情，由用户决定是否继续。话术："⚠️ 发现重大风险信号，建议你在投递/入职前认真考虑以下问题..."

### 🟡 需关注（任一触发，但不满足高风险条件）

| 触发条件 | 所属维度 |
|---------|---------|
| 近1年劳动仲裁 1-2起 | Step 3 |
| 法人近3年变更 ≥2次 | Step 1/5 |
| 参保人数在宣称员工数 50%-70% 之间 | Step 4 |
| 公司规模近1年明显收缩（参保人数下降>20%） | Step 4 |
| 财务数据存在异常波动（营收年变动>50%） | Step 6 |
| 大量分支机构注销（≥3家） | Step 7 |
| 未按时公示年报 | Step 7 |
| 有行政处罚记录（非最严重级别） | Step 3/7 |

> **🟡 动作规则**：在投递建议中明确提醒，标注具体需关注的事项，建议用户在面试/入职前确认细节。

### 🟢 低风险（以上所有条件均未触发）

> 标注"✅ 未发现明显风险信号"，增强用户入职信心。但需注明"数据来源为公开工商信息，可能存在延迟或遗漏"。

## 背调报告输出格式（🆕 扩增为10章节）

完成背调后，输出结构化「企业背调报告」，包含以下10个章节：

1. **📋 基础信息**：公司全称、成立时间、注册资本（实缴/认缴）、经营状态、法定代表人、所属行业、主营业务、人员规模、参保人数
2. **🔍 法人/高管征信评估**（🆕 独立章节）：法定代表人失信/限消/被执行状态、核心高管信用概况、高管曾任职企业风险、综合诚信评级
3. **⚖️ 劳动仲裁与欠薪违法**（🆕 独立章节）：近1年/累计劳动仲裁数量、案件类型分布、原告方分析、是否存在欠薪诉讼、是否在"欠薪黑名单"中
4. **💰 薪资发放风险评估**（🆕 独立章节）：资金链健康度（实缴比例/财务数据/参保趋势）、发薪稳定性交叉验证、社保公积金合规性、裁员/业务收缩信号、综合发薪风险评级
5. **📊 司法风险全景**：经济纠纷、行政处罚、税务异常
6. **🔗 股权架构审查**：实际控制人、受益所有人、法人变更次数、股权出质/冻结
7. **🏢 行业竞争力评估**：行业地位、核心产品、同行对比、融资动态、上市状态
8. **⚠️ 其他异常信号**：工商变更、分支机构、年报公示、联系方式
9. **🎯 综合风险评级**：🟢低风险 / 🟡需关注 / 🔴高风险，附带详细理由
10. **📌 求职者必知清单**：5-7条入职前必须了解/核实的事项，每条对应具体风险点

### 多公司对比模式（🆕 场景二专用）

当为多公司（场景二的匹配推荐或用户同时查询多家）做背调时，额外输出：

- **背调对比总览表**：各公司在薪资风险/劳动仲裁/法人征信/经营健康度四个核心维度的一览对比
- **投递优先级建议**：基于背调结果+岗位匹配度，给出投递优先级排序

---

## 背调报告输出方式

1. **对话内输出**：在对话中直接输出结构化背调报告（使用上述10章节格式）
2. **HTML文件输出**：生成为排版专业的HTML文件，命名规则：
   - 单公司：`背调报告_[公司名称].html`
   - 多公司对比：`背调报告_多公司对比_[日期].html`
3. **重大风险即时预警**：🔴高风险信号发现时，**必须立即中断流程，优先告知用户**

## 企查查MCP调用指南

| 背调维度 | MCP方法 | 关键参数/注意 |
|---------|---------|-------------|
| 锁定公司 | `get_company_by_query` | 用公司全称精确搜索，模糊搜索可能有同名公司 |
| 工商基础 | `get_company_registration_info` | 注册资本看**实缴**而非认缴；经营状态看是否存续 |
| 公司画像 | `get_company_profile` | 重点关注业务模式、行业标签、人员规模、**参保人数** |
| 高管名单 | `get_key_personnel` | 逐个核实法定代表人和董监高，不要只看头衔 |
| 诚信验证 | `verify_company_accuracy` | 交叉验证信息一致性，不一致为异常信号 |
| 股东结构 | `get_shareholder_info` | 关注持股比例、股东类型（自然人/法人/外资） |
| 实际控制人 | `get_actual_controller` | 穿透表层股东，识别真正控制人 |
| 受益所有人 | `get_beneficial_owners` | 识别UBO，判断是否有隐匿实控 |
| 对外投资 | `get_external_investments` | 间接判断资金实力和业务版图 |
| 财务数据 | `get_financial_data` | 营收/利润/纳税，重点关注趋势而非绝对值 |
| 上市信息 | `get_listing_info` | 上市公司信息更透明，未上市需更多维度验证 |
| 工商变更 | `get_change_records` | 频繁变更为异常信号，关注法人/经营范围/地址变更 |
| 分支机构 | `get_branches` | 大规模注销=业务收缩，新增=扩张 |
| 联系方式 | `get_contact_info` | 空号/停机/地址异常为风险信号 |
| 年报 | `get_annual_reports` | 未按时公示=管理不规范，公示内容可验证经营数据 |
| 税务信息 | `get_tax_invoice_info` | 非正常户/欠税/虚开发票为严重风险 |
| 变更记录 | `get_change_records` | 全面工商变更历史 |
