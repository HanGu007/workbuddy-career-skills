# 简历专属诊断

> 本 Skill 为「职途星」专家的核心能力模块，用于在修改简历前进行专属诊断。

## 触发时机

当用户提供了简历内容（文字粘贴或文件上传），且在动手修改之前，必须先执行本诊断流程。

当用户说"帮我诊断简历"/"简历诊断"/"帮我分析这份简历"/"出个诊断报告"或类似表述时，执行完整诊断流程，**Step 4 生成 HTML 诊断报告为必做项——不论用户是否明确说"出报告"，只要触发"简历诊断"意图就必须生成 HTML 报告。HTML 诊断报告是诊断环节的唯一最终交付物，严禁仅输出纯文本诊断而跳过 HTML 报告。**

## 诊断流程

### Step 1：简历内容识别（多策略分层识别）

收到简历后，必须采用**多策略分层识别**方式，确保文字内容和图片元素均不遗漏。

#### 策略一：模型多模态直读（首要策略）

**所有文件类型统一使用 Read 工具直接读取**，Read 工具具备多模态能力，可同时获取文字内容和图片中的视觉信息。

- 用户文字粘贴简历 → 直接阅读理解，无需任何工具
- 用户上传文档文件（docx/pdf/jpg/png等） → 使用 Read 工具直接读取，模型凭借多模态能力同时理解文字和视觉内容

#### 策略二：图片元素专项提取（关键补充）

PDF/文档中的图片元素往往携带重要信息，纯文字提取会遗漏。识别时必须**主动关注并提取**以下视觉元素：

| 图片类型 | 可能携带的信息 | 提取要点 |
|---------|-------------|---------|
| 🪪 证件照/形象照 | 形象展示、职业感判断 | 记录是否存在，评估是否专业得体 |
| 🔏 图章/印章 | 证书真伪、官方认证 | 提取印章文字内容（如"XX大学学位证专用章"），判断认证类型 |
| 📊 截图/数据图表 | 业绩数据、排名证明 | 提取图表中的关键数字和趋势描述 |
| 🏆 证书/奖状图片 | 资质证明、荣誉记录 | 提取证书名称、颁发机构、有效期等文字 |
| 🎨 排版截图/设计元素 | 简历设计风格、创意展示 | 评估视觉设计水平和排版专业度 |
| 📋 表格截图 | 结构化数据（成绩单、项目列表等） | 提取表格中的关键数据项 |
| 🗺️ 其他图片 | 地理位置、架构图、流程图等 | 提取图中核心信息并用文字描述 |

**识别流程**：
1. 使用 Read 工具读取文件 → 获取文字层内容 + 视觉层内容
2. 检查识别结果中是否包含图片元素描述
3. 如有图片元素 → 按"图片元素专项提取"表格逐一提取信息
4. 将图片提取的信息与文字内容**合并整合**，形成完整简历画像

#### 策略三：信息遗漏检测与补充追问

完成识别后，执行**信息完整性自检**，对比文字层和视觉层的提取结果：

```
🔍 简历识别完整性自检：
- [ ] 文字层：姓名/联系方式/教育/工作经历/技能 → 是否完整？
- [ ] 视觉层：图章/截图/照片/证书/表格 → 是否有图片未被识别？
- [ ] 交叉验证：文字层提到的内容（如"获XX证书"）是否有对应图片佐证？
- [ ] 矛盾检测：文字描述与图片信息是否一致？
```

**信息遗漏处理**：
- 发现图片元素未提取到有效信息 → 向用户说明"我在你的简历中发现[图章/截图/证书]图片，但无法完整识别其中的内容，能否告诉我：[具体问题]？"
- 文字层与图片层信息矛盾 → 以图片为准，向用户确认
- 识别结果不完整或格式混乱 → 主动追问用户补充，不得因格式问题跳过

#### 识别结果要求

识别结果应包含完整的简历结构化信息：
- 姓名、联系方式、教育背景、工作经历、技能、项目经历等（文字层）
- 图片元素提取信息汇总（视觉层）：存在哪些图片、各图片携带的关键信息
- 信息完整性评估：是否有遗漏或矛盾

#### ⚠️ 识别纪律

- **优先使用 Read 工具的多模态能力**，它可以在一次读取中同时处理文字和图片
- **禁止调用外部OCR API或付费文档解析服务**做简历识别
- **允许使用内置工具**（如 Read、markitdown 等WorkBuddy内置能力）读取文件内容
- 如果模型通过Read工具仍无法识别图片中的文字（如图章文字模糊），**向用户追问而非猜测**

### Step 2：四维度专属提取

从简历中自动提取以下四个维度的信息，**用一段话告诉用户"你读到的是什么样的人"**：

1. **人物画像**：大概是什么行业、什么岗位方向
2. **经验段位**：工作年限和经验段位（应届/1-3年新人/3-5年中级/5-10年资深/10年+管理层）
3. **三大具体问题**（必须具体，禁止说通用废话）：
   - ❌ 错误示范："缺乏量化"
   - ✅ 正确示范："第2段经历只写了'参与XX项目'，完全没说这个项目多大规模、你在里面的具体贡献是什么、最终结果怎样"
   - ❌ 错误示范："描述太弱"
   - ✅ 正确示范："第3条'负责客户对接'——HR看到这一句只知道你和客户接触过，不知道你接了多少个客户、解决了什么问题、带来了什么结果"
4. **被埋没的亮点**：简历里没有被充分展示的潜在优势——可能是某段经历的规模、某项技能的含金量、或者用户不觉得重要但其实很有竞争力的经历

> **注意**：这一步是针对这份简历的专属诊断，不是通用建议清单。

### Step 3：输出诊断结果

向用户输出格式化的诊断结果，让用户清楚知道"你读到的是什么样的人"以及"这份简历的核心问题在哪里"。

### Step 4：生成 HTML 诊断报告（✅ 必做 — 诊断的唯一最终交付物）

**本步骤为必做项**：只要触发了简历诊断流程（Step 1-3），Step 4 必须执行，不允许跳过。

#### 报告规格

- **尺寸**：A4 纵向（210mm × 297mm），**内容严格控制在单页**
- **格式**：独立HTML文件，内嵌CSS样式，可直接打印或导出PDF
- **命名规则**：`简历诊断报告_[姓名]_[日期].html`，保存到工作目录

#### 报告内容结构（7大模块，必须全部包含）

| # | 模块 | 内容要求 | 版式 |
|---|------|---------|------|
| 1 | 📋 报告头部 | 标题"简历诊断报告"、姓名、诊断日期、经验段位标签 | 顶部横幅，居中 |
| 2 | 🧑 人物画像 | 一句话行业+岗位定位，附"HR第一眼印象"1句话 | 摘要卡片，紧凑排版 |
| 3 | 📊 四维评分 | 量化/专属性/关键词/排版 四项（0-10分），每项配15字内评语 | 横向4个评分卡片，配颜色进度条 |
| 4 | ⚠️ 三大具体问题 | 每个问题：①定位到第X条经历 ②问题原句引用 ③为什么是问题（HR视角） ④具体怎么改（给出改写方向） ⑤严重程度标签（🔴严重/🟠中等/🟡轻微） | 竖向列表，左侧编号+颜色边框 |
| 5 | 💎 被埋没的亮点 | 2-3条：①原简历中的位置 ②为什么有价值 ③改写建议（如果改写成X，HR会看到Y） | 列表形式，带高亮标记 |
| 6 | 🎯 优化优先级建议 | 按紧急程度排序的3-5条行动建议，每条包含：优先级编号 + 操作指令（不是"建议你XX"而是"把第X条改成XX"） | 编号列表，🔴🟠🟡优先级色标 |
| 7 | 📌 报告页脚 | "职途星 · AI简历诊断" + 免责声明（一行小字） | 底部居中 |

#### 🔥 诊断报告干货质量铁律（最高优先级）

> **每一份诊断报告必须是"用户拿到就能照着改"的操作手册，而不是"看着有道理但不知道怎么做"的评论文章。**

1. **杜绝通用废话**：禁止出现"缺乏量化""描述太弱""排版不够专业"等不指向具体位置的评语。每一条诊断必须引用简历原文片段。
2. **每条问题必带改写方向**："把第X条经历的'负责客户对接'改成：'对接XX个客户，主导X个关键项目，客户续约率从X%提升至Y%'（用户自行填入真实数据）"——不能只批评不给方案。
3. **HR视角代入**：每个问题/建议必须解释"HR看到这一条时会怎么想"（如"HR看到'负责日常运营'等于什么都没看到，因为100份简历里有90份都这么写"）。
4. **内容密度要求**：模块间距不超过12px，行高1.3-1.5，字号在可读范围内尽量压缩（正文11-12px，标题14-18px），用紧凑排版替代松散发散布局，确保单页承载最大信息量。但不允许牺牲可读性——所有文字必须清晰可辨。
5. **行动指令化**：优化建议用祈使句（"在第二条经历前加一句行业定位"），不用疑问句或建议句（"你可以考虑..."）。

#### 视觉规范

- **配色方案**：主色 #2563EB（蓝色），辅色 #F59E0B（金色），背景 #FFFFFF
- **评分卡片**：每个卡片配渐变色背景条表示分数（低于6分红色预警，6-7分黄色，8分以上绿色）
- **问题严重程度**：🔴 严重 → 红色左边框(#EF4444)，🟠 中等 → 橙色左边框(#F59E0B)，🟡 轻微 → 黄色左边框(#EAB308)
- **字体**：标题 14-18px，正文 11-12px，辅助文字 10px
- **间距**：模块间 margin-bottom 8-12px，卡片内 padding 8-12px；严禁大片空白（任何 > 30px 纯色空白区域视为不合格）
- **单页保障**：全部模块高度总和不超过 270mm（含上下页边距各 10mm），如内容溢出优先缩小辅助文字/减少padding，其次精简描述

#### 输出流程

1. 完成 Step 1-3 的诊断分析
2. 将诊断结果按报告结构组织为结构化数据
3. 生成 HTML 诊断报告文件到工作目录
4. 使用 `preview_url` 预览报告，确认单页不溢出
5. 执行交付物三检（存在性→内容完整性→deliver_attachments）
6. 在聊天中输出「📋 简历诊断报告已生成，包含：人物画像 + 四维评分 + 三大问题(含改写方向) + 被埋没亮点(含改写建议) + 优化行动指令」

## 诊断质量要求

- 三大问题必须**具体到第X条经历+引用原文片段+给出改写方向**，不说通用废话
- 被埋没的亮点要说出**为什么有价值 + 如果改写会带来什么效果**（"如果写成XX，HR会看到YY"）
- 人物画像要包含**行业+岗位方向**，让用户知道你读准了
- 经验段位要影响后续改写策略（段位匹配规则在 resume-rewrite Skill 中）
- **HTML报告是必出产物**：任何"简历诊断"触发都必须生成HTML报告，严禁仅输出纯文本
- **诊断报告必须有改写方向**：每个问题和亮点必须附带具体的改写建议，让用户拿到报告就能动手改
